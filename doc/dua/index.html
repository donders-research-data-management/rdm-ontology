Data Use Agreements

To share data, analysis pipelines and/or software in a "data sharing collection", the collection manager needs to select a Data Use Agreement (DUA) before publishing a collection. A DUA specifies the conditions and restrictions under which data is shared. For instance, attribution may be required in future publications using the data, or re-sharing of the data may be restricted. In some cases, restrictions are necessary to prevent participant disclosure. Given applicable legislation (for instance GDPR) it is important to select the correct DUA before sharing data.

Several DUAs are described below for sharing (1) potentially identifiable data and (2) non-human/non-identifiable human data. In case your data contains potentially sensitive information(i.e. information that is competition-sensitive or confidential for individuals, communities, employers) or in case the data has been acquired in an industry-sponsored project, you must contact your data steward and the RU legal department to make a tailored agreement.

By selecting a DUA, the collection manager also (implicitly) determines the required access level for external researchers. Specifically, published DSCs can be accessed either (1) following authentication, or (2) anonymously, and the selected DUA determines this, see below.



1. Sharing potentially identifiable data

In order to protect the privacy of the participants, the Donders Institute / RU have developed a default DUA for sharing potentially identifiable human data. This DUA must be used when data is potentially identifiable:

RU-HD-1.0 Radboud University - Radboud University - Human Data 1.0

When selecting this DUA, external researchers can only access published data after authentication and explicitly agreeing to this DUA. The collection manager can identify external collection users,as they will be added as "viewer" to the collection.

What is potentially identifiable data?

Some of the data that we acquire is directly identifiable, such as MR-scans, audio, video and photo data. This data must be de-identified before sharing (see here).

De-identified (audio, video, photo or MRI) data is regarded as potentially identifiable. Also, a combination of background or demographic data could make a dataset potentially identifiable (e.g. gender, age and disease). You need to assess when this is the case. Contact the data steward in case of doubt.

In exceptional cases in which de-identifying would make the data useless AND for which participants have given consent to share direct identifiable data, it can be allowed to share identifiable data. In these cases, approval by the data steward is required.



2. Sharing non-human or non-identifiable human data with anonymous users

If you share non-identifiable data, we recommend a more open access, 'general' license. If your non-identifiable data do not require a RU agreement or were acquired outside Radboud University, you can use one of the OpenDataCommons or CreativeCommons licenses (see below). See here to help make a decision.

Users can anonymously access data publications shared under any of these licenses and (implicitly) accept the terms by downloading the data. Collection managers cannot identify external users of the collection in any way. Data published under any of these licenses must be 100% anonymized (see here).
